# HEARTBEAT.md - 心跳任务清单

## 每次心跳必做

- [ ] 检查并记录本次会话的重要对话到 `memory/YYYY-MM-DD.md`
- [ ] 查看是否有用户待回复的消息或问题

## 每天轮换检查（分散 API 调用）

### 早晨 (09:00) 重点
- [ ] 查看系统状态（Gateway、服务运行状态）
- [ ] 检查工作区 git 状态（是否有未提交的变更）

### 下午 (14:00) 重点
- [ ] 检查项目进度（如有正在进行的任务）
- [ ] 查看是否有错误日志需要关注

### 晚上 (20:00) 重点
- [ ] 整理当天 memory 记录
- [ ] 回顾是否有重要内容需要提炼到 MEMORY.md
- [ ] 生成简单的当日总结
- [ ] **在 journal 仓库写当天日记** (`/mnt/e/claw/journal/diary/YYYY-MM-DD.md`)

## 原则

1. **主动记录** - 对话中有价值的内容及时写入 memory
2. **避免重复** - 已检查过的事项不要重复汇报
3. **有异常才说** - 正常情况回复 HEARTBEAT_OK，有问题再详细汇报
4. **API 节约** - 分散检查任务，不要每次心跳都调用所有外部 API
